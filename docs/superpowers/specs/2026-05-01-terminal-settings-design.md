# Terminal Settings GUI + Theme Picker — Design Spec

**Status:** Draft (awaiting user review)
**Date:** 2026-05-01
**Scope:** macOS app (`apps/macos/`) only.

---

## 1. Goal

Replace the current "⌘, opens TOML in Finder" behavior with a proper macOS Preferences window covering Font, Cursor, Bell, Scrollback, Window, and Theme settings. Add a theme picker (Ghostty's full theme list, with 9 curated favorites). Support **per-host theme override** (other fields are global only in v1).

## 2. Architecture

### 2.1 Persistence: Caterm-owned plist + rendered managed snapshot

```
[Ghostty defaults]
        ↓
[managed snapshot TOML]   ← auto-generated by Caterm from settings.plist
        ↓
[user config TOML]         ← unchanged; power-user escape hatch always wins
```

- New file: `~/Library/Application Support/Caterm/settings.plist` (Codable, Caterm-owned).
- Existing `ConfigStore` already manages the snapshot layer between Ghostty defaults and user config. We extend it to render `settings.plist` into TOML on every write.
- The user TOML at `~/Library/Application Support/Caterm/config` is **never modified** by the GUI.

### 2.2 Settings schema (Codable)

```swift
public struct CatermSettings: Codable, Equatable {
    public var version: Int                       // schema version, starts at 1
    public var revision: String                   // ULID-like token; bumps on any change
    public var global: PartialSettings
    public var hostOverrides: [HostId: PartialSettings]
}

public struct PartialSettings: Codable, Equatable {
    public var fontFamily: String?               // system monospaced fonts only
    public var fontSize: Int?                    // 8..32
    public var lineHeight: Double?               // 0.8..2.0  → adjust-cell-height
    public var cursorStyle: CursorStyle?         // .block | .bar | .underline
    public var cursorBlink: Bool?
    public var bell: BellMode?                   // .none | .audio | .visual | .both
    public var scrollbackLines: Int?             // default 10000
    public var windowOpacity: Double?            // 0.7..1.0
    public var windowPaddingX: Int?
    public var windowPaddingY: Int?
    public var theme: String?                    // theme name (matches Ghostty's list)
}
```

In v1 only `theme` is read from `hostOverrides[hostId]`; other fields are **global only** (the data model permits future expansion without migration).

### 2.3 Live reload pipeline

```
User edits in Preferences
   ↓
SettingsStore.update(...) writes plist + bumps revision
   ↓
ConfigStore.renderManagedSnapshot(from: settings)   → writes managed TOML
   ↓
NotificationCenter post .catermSettingsChanged(revision)
   ↓
For each GhosttySurfaceNSView:
   - resolve effective theme (hostOverrides[hostId].theme ?? global.theme)
   - call ghostty_surface_reload_config(surface)
```

Fallback: if `ghostty_surface_reload_config` returns failure for a particular field (e.g. font-family on certain Ghostty versions), the surface is flagged with "next reconnect to apply" — no app restart.

## 3. UI Components

### 3.1 Preferences window (`PreferencesWindowController`)

Standard NSWindow + Toolbar with tabs:

```
┌───────────────────────────────────────────┐
│ ⚙️ General  📺 Terminal  🎨 Themes  ☁ Sync │
├───────────────────────────────────────────┤
│                                           │
│         (active tab content)              │
│                                           │
└───────────────────────────────────────────┘
```

- Replaces `AppDelegate.openSettings()` (currently opens Finder).
- ⌘, opens this window. ⌘W closes it.
- Window saves frame to UserDefaults (`PreferencesWindowFrame`).
- Each tab is a SwiftUI view embedded via `NSHostingController`.

- Source: `ghostty +list-themes` parsed at app start; cached in memory.

- Search filters the "All Themes" grid (favorites always visible).

### 3.5 Tab: Sync (existing `SyncSettingsView`)

Migrate the current sheet into this tab. Functionality unchanged.

### 3.6 Per-host theme override UI

In existing `HostFormView`, append a section:

```
Theme Override
  [Use global ▾]          ← dropdown: "Use global" + theme list
```

When set to anything other than "Use global", the host stores `hostOverrides[hostId].theme = "..."`. Removed/cleared → entry deleted from `hostOverrides`.

**Effect timing (per Q4a):** new tabs to that host pick up the override immediately. Existing tabs keep their current theme until reconnect.

## 5. Data Flow Detail

### 5.1 Read path (boot)

1. `SettingsStore.load()` reads `settings.plist`. If absent → seed defaults (matches current managed snapshot defaults).
2. `ConfigStore.renderManagedSnapshot(from: settings.global)` writes the managed TOML.
3. Ghostty reads its full chain (defaults → managed → user TOML).

### 5.2 Write path (user edits in Preferences)

1. UI action mutates `SettingsStore.settings.global.<field>`.
2. Store debounces 200ms then:
   - Bumps `revision` (ULID)
   - Persists plist atomically (`.write(to:options:.atomic)`)
   - Triggers `ConfigStore.renderManagedSnapshot(from: ...)`
   - Posts `.catermSettingsChanged` notification
3. Listening surfaces call `ghostty_surface_reload_config()`.

### 5.3 Per-host theme resolution (new tab)

```swift
let effectiveTheme = settings.hostOverrides[hostId]?.theme ?? settings.global.theme ?? "Catppuccin Mocha"
```

This resolution happens **at surface creation time**. `ConfigStore.renderManagedSnapshot` always renders the global theme into the snapshot; per-host override is applied via a **surface-local config patch** (`ghostty_config_load_string` over the loaded config) when the surface is constructed.

## 6. Field → TOML mapping

| Schema field | Ghostty TOML key |
|---|---|
| `fontFamily` | `font-family` |
| `fontSize` | `font-size` |
| `lineHeight` | `adjust-cell-height` (e.g. `10%` for 1.1) |
| `cursorStyle` | `cursor-style` |
| `cursorBlink` | `cursor-style-blink` |
| `bell == .audio` | `audible-bell = true` |
| `bell == .visual` | `visual-bell = true` |
| `bell == .both` | both true |
| `scrollbackLines` | `scrollback-limit` |
| `windowOpacity` | `background-opacity` |
| `windowPaddingX/Y` | `window-padding-x` / `window-padding-y` |
| `theme` | `theme` |

`SettingsRenderer` emits a header comment: `# managed by Caterm — do not edit; use Caterm Preferences (⌘,)`.

## 7. Error Handling

| Failure | Behavior |
|---|---|
| `settings.plist` corrupted | Quarantine to `settings.plist.broken-<timestamp>`; seed defaults; surface a non-modal alert on next launch |
| `ghostty +list-themes` fails | Fall back to bundled 9-favorites list; show banner in Themes tab |
| `ghostty_surface_reload_config` returns error | Mark surface "needs reconnect"; show subtle indicator in tab |
| Invalid font-family typed (shouldn't happen with dropdown but defensive) | Renderer omits the line; managed snapshot stays valid |
| User TOML has same key as managed | User TOML wins (correct & intentional) |

## 8. Migration

- First launch with this version: no `settings.plist` → seed defaults that **match the current managed snapshot baked into ConfigStore today** (SF Mono, size 13, Catppuccin Mocha, block cursor, etc.). Users see no visual change.
- The hard-coded managed snapshot constants in `ConfigStore` are deleted; replaced by `SettingsRenderer.render(from: defaultSettings)`.

## 9. Testing

### 9.1 Unit tests

- **`SettingsStoreTests`**
  - plist round-trip (write → read → equal)
  - corrupted plist → quarantine + defaults
  - debounce coalesces rapid edits into one write

- **`SettingsRendererTests`**
  - each field renders to expected TOML line
  - `lineHeight = 1.1` → `adjust-cell-height = 10%`
  - `bell = .both` → both flags true
  - `theme` with quotes/spaces is escaped
  - empty `PartialSettings` → empty TOML body (still has header comment)

- **`ThemeCatalogTests`**
  - parses `ghostty +list-themes` output
  - falls back to bundled list on parse failure

- **`ConfigStoreTests` (existing, extend)**
  - render produces a snapshot file with expected content
  - per-host override patch path produces correct config string

### 9.2 Manual smoke (`apps/macos/Manual/settings-smoke.md`)

1. ⌘, opens Preferences (not Finder)
2. Change font size → all open tabs reflow live
3. Change cursor style → live update
4. Change theme → live update
5. Close & reopen app → settings persist
6. Edit user TOML to override one field → user value wins after restart
7. Click "Edit Advanced Config" → user TOML opens in Finder
8. Set per-host theme override on host A → connect to A → tab uses override
9. Connect to host B (no override) → tab uses global theme
10. Quit while typing in stepper → no data loss (debounce flushes on quit)

## 10. Out of Scope (v2)

- letter-spacing
- inactive cursor style
- per-ANSI-color overrides
- keybinding editor
- shell selection
- per-host overrides for non-theme fields
- cloud sync (Y2 deferred — requires server schema + conflict policy)
- light/dark auto-switch on system appearance
- import/export settings file
