# Task v1.3 — Sync Failure Visibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make persistent auto-sync failure **visible** so the user does not need to manually press "Sync Now" to discover that sync has been silently broken for 15+ minutes. Two surfaces: an always-on red "Sync failing since …" line in `SyncSettingsView` (gated by `TimelineView` re-render), and an opt-in `UNUserNotification` on the `normal → failing` edge for auto-cycles only.

**Architecture:** Three coordinated changes, no new module. (1) `SyncPreferences` gains a `notifyOnFailureEnabled` opt-in toggle (default OFF). (2) `HostSyncStore` gains three `@Published` props (`lastSyncAttemptedAt`, `lastSyncErrorKind`, `failingSince`), a private `wasFailing` edge tracker, a `notificationCenter:` init param, a public `clearAuthError()` method, a free `syncFailureState(...)` derivation, and a `do/catch` wrap around `performSync()`'s body with a `CancellationError` carve-out. `periodicInterval` flips from `private let` to `public let` so the view layer can read it. (3) `SyncSettingsView` extends `accountState(...)` with `lastSyncErrorKind`, adds a "Notify when sync fails" Toggle, and renders red failing text + a "Turn on Notify…" hint inside the existing `TimelineView(.periodic(by: 30))` block. Sign-out and `onSignedIn` paths invoke `clearAuthError()`.

**Tech Stack:** Swift 6 (`@MainActor` isolation), SwiftPM, Combine (`@Published`, `Timer.publish`, `NotificationCenter` publisher), SwiftUI (`@StateObject`, `@ObservedObject`, `TimelineView(.periodic)`), `UserNotifications` (`UNUserNotificationCenter`, `UNNotificationRequest`), `UserDefaults` (with `suiteName:` for test isolation), XCTest.

**Spec:** `docs/superpowers/specs/2026-04-28-task-v1.3-failure-visibility-design.md` — re-read before starting if any architectural question arises. Locked decisions are in §10 (21 items).

```swift
public enum SyncErrorKind: Equatable, Sendable {
    case auth     // ServerSyncError 401-shaped: .http(401, _) / .orpc(_, 401, _) / .authFailed / .notSignedIn
    case other    // network / 5xx / decode error / etc.
}

public enum SyncFailureState: Equatable {
    case normal
    case failing(reason: SyncErrorKind, since: Date)
}

public protocol NotificationDelivering: Sendable {
    func add(_ request: UNNotificationRequest) async throws
}

public struct LiveNotificationCenter: NotificationDelivering {
    public init() {}
    public func add(_ request: UNNotificationRequest) async throws {
        try await UNUserNotificationCenter.current().add(request)
    }
}

/// Pure derivation of the failing-state surface. No `self`, no actor — testable
/// in isolation. Spec §2.2.4.
///
/// `since` here is the *most recent* attempted timestamp; the store's
/// `failingSince` (= the FIRST failure of a run) is supplied to the view layer
/// separately. This function operates on inputs alone.
public func syncFailureState(
    now: Date,
    lastSyncedAt: Date?,
    lastSyncAttemptedAt: Date?,
    lastSyncErrorKind: SyncErrorKind?,
    periodicSyncEnabled: Bool,
    failingThreshold: TimeInterval
) -> SyncFailureState {
    guard periodicSyncEnabled else { return .normal }
    guard let attempted = lastSyncAttemptedAt else { return .normal }
    guard let succeeded = lastSyncedAt else { return .normal }
    guard attempted > succeeded else { return .normal }
    guard now.timeIntervalSince(succeeded) > failingThreshold else { return .normal }
    let reason = lastSyncErrorKind ?? .other
    return .failing(reason: reason, since: attempted)
}
```

- [ ] **Step 1: Rewrite `performSync()` with do/catch + state writes**

Find the existing `private func performSync() async throws` (HostSyncStore.swift:188-202). **Replace the entire function body** with:

```swift
    private func performSync() async throws {
        // Head: record attempt (covers both manual and auto paths). The
        // catch path uses lastSyncAttemptedAt as the failingSince fallback,
        // so this MUST run before the do block.
        let attempted = Date()
        lastSyncAttemptedAt = attempted
        userDefaults.set(attempted, forKey: Self.lastSyncAttemptedAtKey)

        do {
            let remote = try await client.listHosts()
            try Task.checkCancellation()
            let ops = HostSyncReconciler.reconcile(local: sessionStore.hosts,
                                                    remote: remote)
            for op in ops {
                try Task.checkCancellation()
                try await apply(op)
            }
            // Success — only after op loop finishes without throwing.
            // Co-locates with v1.2 lastSyncedAt write; same all-or-nothing
            // gating (partial-apply leaves lastSyncedAt untouched).
            let now = Date()
            lastSyncedAt = now
            userDefaults.set(now, forKey: Self.lastSyncedAtKey)
            lastSyncErrorKind = nil
            wasFailing = false
            failingSince = nil
        } catch is CancellationError {
            // Chained cancel-and-drain (HostSyncStore.swift:173-184) replaces
            // this Task with a newer one; treat as benign re-arm, NOT a
            // failure. Don't classify, don't fire notification, don't update
            // wasFailing. The pattern-match covers all three cancel sites
            // (listHosts / checkCancellation / apply).
            throw CancellationError()
        } catch {
            lastSyncErrorKind = classifySyncError(error)
            let nowFailing = isCurrentlyFailing()
            if !wasFailing && nowFailing {
                failingSince = lastSyncAttemptedAt
                if !manualInProgress && preferences.notifyOnFailureEnabled {
                    await fireFailureNotification()
                }
            }
            wasFailing = nowFailing
            throw error
        }
    }
```

> Spec §2.2.5 / §4.7. Note the order inside the failure branch: write `failingSince` and fire notification BEFORE updating `wasFailing` (so the edge gate `!wasFailing && nowFailing` reads the prior value), then the `wasFailing = nowFailing` write at the end of the branch caches the new state for the *next* cycle's edge gate.

- [ ] **Step 2: Add the three private helpers**

```swift
    private func classifySyncError(_ error: Error) -> SyncErrorKind {
        guard let err = error as? ServerSyncError else { return .other }
        switch err {
        case .http(status: 401, _):    return .auth
        case .orpc(_, status: 401, _): return .auth   // oRPC envelope wraps 401, NOT .http
        case .authFailed:              return .auth
        case .notSignedIn:             return .auth
        default:                       return .other
        }
    }

    private func isCurrentlyFailing() -> Bool {
        if case .failing = syncFailureState(
            now: Date(),
            lastSyncedAt: lastSyncedAt,
            lastSyncAttemptedAt: lastSyncAttemptedAt,
            lastSyncErrorKind: lastSyncErrorKind,
            periodicSyncEnabled: preferences.periodicSyncEnabled,
            failingThreshold: periodicInterval
        ) {
            return true
        }
        return false
    }

    private func fireFailureNotification() async {
        let content = UNMutableNotificationContent()
        content.title = "Caterm sync is failing"
        content.body = "Click Sync Now in Sync Settings, or check your connection."
        let request = UNNotificationRequest(
            identifier: "caterm.sync.failing.\(UUID().uuidString)",
            content: content,
            trigger: nil
        )
        try? await notificationCenter.add(request)
    }
```

```swift
    private func handlePeriodicEnabled(_ enabled: Bool) {
        periodicTimerCancellable?.cancel()
        periodicTimerCancellable = nil
        guard enabled else {
            // v1.3 add — clear failure state so re-enable starts clean.
            wasFailing = false
            failingSince = nil
            lastSyncErrorKind = nil
            // P1 round-4 fix: also clear lastSyncAttemptedAt and its
            // persisted key. Otherwise a stale `attempted > succeeded`
            // makes syncFailureState return .failing immediately on
            // toggle-on, before any new sync cycle has run — contradicts
            // the "fresh start" semantic users expect from off → on.
            // lastSyncedAt is intentionally NOT cleared (preserves the
            // "Last sync: …" relative phrase across the off-state).
            lastSyncAttemptedAt = nil
            userDefaults.removeObject(forKey: Self.lastSyncAttemptedAtKey)
            return
        }
        periodicTimerCancellable = Timer.publish(every: periodicInterval,
                                                  on: .main,
                                                  in: .common)
            .autoconnect()
            .sink { [weak self] _ in self?.scheduleAutoSync() }
    }
```

> Spec §2.2.7 / decision #9. Cleanup MUST live inside the `else { return }` branch — the sink fires synchronously on init with `enabled = true`, then on every value change; clearing on `enabled = true` would erase legitimate state.
